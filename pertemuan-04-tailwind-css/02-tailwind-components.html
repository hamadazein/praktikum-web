Komponen Tailwind CSS

Membangun komponen UI yang indah dengan utility classes

File ini mengumpulkan contoh komponen Tailwind CSS yang sering dipakai dalam proyek web modern. Setiap komponen dilengkapi penjelasan, potongan kode, dan tips implementasi agar Anda memahami utility-first workflow secara praktis.

Pendekatan yang akan kita gunakan mengikuti siklus copy → pahami → modifikasi. Salin kode contoh, pahami struktur dan utility class yang dipakai, lalu modifikasi sesuai kebutuhan aplikasi Anda.

Learning Goals

  • Memahami struktur Tailwind untuk berbagai komponen
  • Menerapkan kombinasi utility class secara efektif & konsisten
  • Mempraktikkan best practice Tailwind pada layout responsive

1. Card Components

Komponen kartu memudahkan kita menampilkan potongan informasi secara terstruktur: gambar, judul, metadata, dan aksi. Tailwind menyediakan utility untuk spacing, bayangan, tipografi, serta efek hover sehingga kartu terasa interaktif tanpa perlu menulis CSS manual.

  • Gunakan rounded, shadow, dan overflow-hidden untuk membentuk kartu yang rapi.
  • Kelompokkan konten utama di dalam container p-6 agar jarak antar elemen konsisten.
  • Tambahkan hover:shadow-xl atau transition untuk memberikan feedback interaksi.
  • Pastikan gambar memakai object-cover agar proporsi tetap bagus di berbagai ukuran kartu.

2. Navigation Components

Navigasi adalah anchor utama pengalaman pengguna. Dengan Tailwind kita bisa mengatur alignment, spacing, dan state (aktif/hover) secara eksplisit sehingga menu mudah dibaca dan responsif. Gabungkan utility seperti flex, justify-between, dan space-x-* untuk mengontrol layout horizontal.

Gunakan breakpoint md: untuk menampilkan menu desktop dan sembunyikan pada mobile dengan hidden md:block.
Pastikan tombol hamburger mobile memiliki area klik besar dan ikon jelas.
Breadcrumb membantu orientasi pengguna. Gunakan inline-flex dan ikon separator untuk menegaskan hierarchy.
Sertakan atribut aria-label agar navigasi mudah diakses pengguna screen reader.

3. Form Components

Formulir membutuhkan perhatian ekstra pada aksesibilitas dan konsistensi spacing. Tailwind memudahkan kita menjaga jarak, warna fokus, serta tipografi label sehingga form terasa nyaman digunakan di perangkat apa pun.

Selalu padukan label dan input agar form ramah aksesibilitas.
Gunakan focus:ring dan focus:border-transparent supaya outline terlihat modern.
Manfaatkan space-y-4 untuk merapikan jarak vertikal antar field.
Pastikan tombol submit memiliki lebar penuh (w-full) pada layar kecil.

4. Interactive Components

Komponen interaktif seperti modal, tabs, dan alert membutuhkan perpaduan utility Tailwind dan sedikit JavaScript untuk mengontrol state. Gunakan utility transition dan animation agar feedback terasa halus, sedangkan state (aktif/tersembunyi) bisa diatur dengan menambah/menhapus class.

Simpan state UI di JavaScript lalu toggle class Tailwind seperti hidden, flex, atau opacity-0.
Manfaatkan transition dan duration-300 untuk animasi masuk/keluar yang natural.
Pastikan modal memiliki backdrop (bg-gray-600/50) dan dapat ditutup dengan klik luar atau tombol escape.
Alert sebaiknya memakai warna kontras dan ikon agar informasi cepat dipahami.

Code Example - Modal & Tabs Toggle:

<!-- Modal Trigger -->
<button id="openModal" class="bg-purple-500 text-white px-4 py-2 rounded">Open Modal</button>

<!-- Modal -->
<div id="modal" class="fixed inset-0 bg-gray-600/60 hidden items-center justify-center">
  <div class="bg-white rounded-lg shadow-xl max-w-md w-full p-6 space-y-4">
    <h3 class="text-lg font-semibold">Modal Title</h3>
    <p class="text-gray-600">Konten modal dengan Tailwind.</p>
    <div class="flex justify-end space-x-3">
      <button id="closeModal" class="bg-gray-200 px-4 py-2 rounded">Cancel</button>
      <button class="bg-purple-500 text-white px-4 py-2 rounded">Confirm</button>
    </div>
  </div>
</div>

<script>
  const modal = document.getElementById('modal');
  document.getElementById('openModal').onclick = () => {
    modal.classList.remove('hidden');
    modal.classList.add('flex');
  };
  document.getElementById('closeModal').onclick = () => {
    modal.classList.add('hidden');
    modal.classList.remove('flex');
  };
</script>

<!-- Tabs -->
<div class="border-b flex">
  <button class="tab-button active px-4 py-2 text-blue-600 border-b-2 border-blue-600" data-tab="tab1">Tab 1</button>
  <button class="tab-button px-4 py-2 text-gray-500" data-tab="tab2">Tab 2</button>
</div>
<div id="tab1" class="tab-content py-4">Konten tab 1</div>
<div id="tab2" class="tab-content hidden py-4">Konten tab 2</div>

<script>
  const buttons = document.querySelectorAll('.tab-button');
  const contents = document.querySelectorAll('.tab-content');
  buttons.forEach(btn => btn.addEventListener('click', () => {
    buttons.forEach(b => b.classList.remove('text-blue-600','border-b-2','border-blue-600','active'));
    contents.forEach(c => c.classList.add('hidden'));
    btn.classList.add('text-blue-600','border-b-2','border-blue-600','active');
    document.getElementById(btn.dataset.tab).classList.remove('hidden');
  }));
</script>

5. Advanced Layout Examples

Layout kompleks seperti dashboard memanfaatkan kombinasi grid dan flex untuk menata kartu statistik, tabel, serta panel aksi. Tailwind mempermudah adaptasi antar breakpoint—kita cukup menambahkan prefix seperti md: atau lg: tanpa menulis media query manual.

Gunakan grid untuk layout makro (kolom kartu) dan flex untuk menyelaraskan isi tiap kartu.
Atur span kolom dengan lg:col-span-2 atau xl:col-span-3 guna menekankan blok penting.
Manfaatkan warna utilitas (bg-blue-500, bg-green-500) sebagai indikator status yang konsisten.
Simpan spacing konsisten dengan p-6/gap-6 agar dashboard rapi dan mudah discan.

Tailwind CSS Best Practices

Terapkan tips berikut untuk menjaga konsistensi, maintainability, dan performa ketika menggunakan Tailwind CSS di proyek profesional.

1. Utility Grouping

Kelompokkan utilitas mengikuti pola layout → spacing → typography → state agar class order mudah dibaca.

2. Responsive First

Mulai dengan style dasar lalu tambahkan breakpoint (sm:, md:, dll) hanya jika diperlukan.

3. Extract Components

Gunakan @apply atau komponen partial bila kombinasi utility mulai panjang dan berulang.

4. Konsisten dengan Design Token

Tetapkan warna & font pada tailwind.config.js sehingga tim memakai nama token yang sama.

5. Optimasi Produksi

Aktifkan fitur content purge agar bundle CSS tetap ramping saat build production.

6. Dokumentasikan Komponen

Catat penggunaan komponen + variasi states (hover/focus) supaya mudah dipakai ulang oleh tim.
